## The Pan workflow

<Steps>
  <Step title="Create a wallet">
    Wallets are the foundation of Pan. Each wallet supports multiple blockchains with a single address, eliminating the need to manage separate addresses per chain.
  </Step>
  
  <Step title="Check balances">
    Query balances across all chains and tokens with a single API call. Pan handles the complexity of querying multiple blockchains and formatting the results.
  </Step>
  
  <Step title="Create an intent">
    Express what you want to achieve (e.g., "lend $1000 USDC") instead of specifying exact transactions. Pan's intelligent system handles the rest.
  </Step>
  
  <Step title="Monitor execution">
    Track intent progress in real-time. Pan provides detailed status updates, execution plans, and comprehensive error reporting.
  </Step>
</Steps>

### Developers and API keys

Developers authenticate using API keys that identify their account. Each developer account has:
- Subscription plan (determines limits)
- Multiple API keys (for different environments)
- Multiple wallets (for different users in their application)

### Wallets

Wallets represent user accounts in your application. Each wallet:
- Has a unique identifier (`walletId`)
- Supports multiple blockchains with a single address
- Can execute multiple intents
- Maintains balances across all supported chains

### Intents

Intents represent high-level financial goals. When you create an intent, Pan:
1. Analyzes the current wallet state
2. Evaluates available DeFi protocols
3. Generates an optimized execution plan
4. Executes the plan automatically

### Execution plans

Execution plans are detailed strategies that describe how Pan will achieve an intent. They include:
- List of operations (swaps, bridges, deposits)
- Order of execution
- Expected costs and timing
- Alternative strategies if needed

## Example flow

Let's walk through a complete example:

<Steps>
  <Step title="Create wallet">
    ```bash
    POST /v1/wallets
    {
      "userId": "alice_123",
      "metadata": {"email": "alice@example.com"}
    }
    ```
    
    Result: Wallet created with address `0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb`
  </Step>
  
  <Step title="Check balance">
    ```bash
    GET /v1/balances/wallet_abc123
    ```
    
    Result: Alice has 500 USDC on Ethereum Sepolia and 500 USDT on Base Sepolia
  </Step>
  
  <Step title="Create intent">
    ```bash

      "walletId": "wallet_abc123",

      "amount": "1000",
      "asset": "USDC",
      "chain": "ethereum-sepolia"
    }
    ```
